Why Isn't My File Being Parsed?

1. Check Radiosonde File Format


File Format 1



Highlighted in green is the correct format for file format 1 where the radiosonde contains the words "Profile Data:", has the variable names in the first line below it, the units in the second line below it, and the data in the third line and onwards. The order of and the amount of variables does not matter since the program will determine that dynamically. The variables that MUST be present however are Time "Time" (or time as "UTC" or "UTC Time"), Temperature "T", Wind Speed "Ws", Wind Direction "Wd", Longitude "Long.", Latitude "Lat.", and Altitude "Alt". If your file contains these variables but with a different abbreviation, please change the abbreviation to the ones stated in quotation marks in the previous sentence.


File Format 2



Highlighted in green is the correct format for file format 2 where the radiosonde contains the variable names in the first line and the data in the second line and onwards. The order of and the amount of variables does not matter since the program will determine that dynamically. The variables that MUST be present however are Time "Time" (or time as "UTC"), Temperature "T", Wind Speed "Ws", Wind Direction "Wd", Longitude "Long.", Latitude "Lat.", and Altitude "Alt". If your file contains these variables but with a different abbreviation, please change the abbreviation to the ones stated in quotation marks in the previous sentence.

2. Check Radiosonde Data


Common Data Issues



- Radiosonde data that prematurely ended such as at 10~11km (the software requires troposphere and stratosphere data to calculate the tropopause and plot the graphs)

- The altitude decreases at unexpected points (the program checks if the weather balloon popped by parsing through altitude values and checking if they are decreasing for at least 5 consecutive data points. The program stops parsing the file at that point to remove data collected while the radiosonde is falling).

- The radiosonde has invalid data symbols (the program currently supports these symbols for invalid data that should be skipped over: "999999", "1000002", "-", "--", "999999.0", "999999.00", "999999.000", '-999.9', and "-9999.9")
